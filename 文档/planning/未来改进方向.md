## 一、图片 PDF 支持（OCR）

### 当前问题

当前系统使用 `PyPDFLoader` 读取 PDF 文件，该库只能提取**文本型 PDF** 中的文字。如果 PDF 是**扫描版**（图片），`PyPDFLoader` 将返回空内容或极少文字。

```python
# 当前实现（document_loader.py:18）
from langchain_community.document_loaders import PyPDFLoader

def load_document(file_path: str) -> List[Document]:
    ext = Path(file_path).suffix.lower()
    if ext == ".pdf":
        loader = PyPDFLoader(file_path)  # ← 只能读取文本型 PDF
    ...
```

### 法律场景影响

法律文档中大量存在扫描版 PDF：
- 历史法律法规（早期数字化版本）
- 裁判文书扫描件
- 合同扫描件
- 公证书、鉴定报告

### 改进方案

#### 方案 A：集成 Unstructured（推荐）

`Unstructured` 库支持多种 PDF 解析模式，包括 OCR：

```python
from langchain_community.document_loaders import UnstructuredFileLoader

loader = UnstructuredFileLoader(
    file_path,
    mode="elements",  # 按元素解析（段落、表格、图片）
    strategy="ocr_only",  # 或 "hi_res"（高精度，需要更多依赖）
)
```

**依赖**：
```bash
uv add unstructured[pdf]  # 基础 PDF 支持
uv add unstructured[ocr]  # OCR 支持（需要 Tesseract）
```

**优点**：
- 统一接口，支持多种格式
- 自动检测文本型/扫描型 PDF
- 支持表格提取

**缺点**：
- 依赖较多（Tesseract、Poppler 等）
- OCR 精度依赖图片质量

#### 方案 B：集成 PaddleOCR

PaddleOCR 是百度开源的 OCR 引擎，中文识别精度高：

```python
from paddleocr import PaddleOCR

ocr = PaddleOCR(use_angle_cls=True, lang="ch")
result = ocr.ocr("scanned_pdf_page.png", cls=True)
```

**优点**：
- 中文识别精度高
- 支持版面分析
- GPU 加速

**缺点**：
- 需要额外安装 PaddlePaddle
- 模型较大（~500MB）

#### 方案 C：混合方案

1. 先用 `PyPDFLoader` 尝试提取文本
2. 如果提取的文本过少（如 < 100 字），判断为扫描版
3. 调用 OCR 引擎处理

```python
def load_pdf_smart(file_path: str) -> List[Document]:
    """智能 PDF 加载：自动检测文本型/扫描型。"""
    # 先尝试文本提取
    loader = PyPDFLoader(file_path)
    docs = loader.load()

    # 检查提取的文本量
    total_text = sum(len(doc.page_content.strip()) for doc in docs)
    if total_text < 100 * len(docs):  # 每页少于 100 字
        # 切换到 OCR 模式
        loader = UnstructuredFileLoader(file_path, strategy="ocr_only")
        docs = loader.load()

    return docs
```

## 二、其他改进方向

### 2.1 向量数据库升级

**当前**：ChromaDB（轻量级，单机）

**可选升级**：
- **Milvus**：分布式向量数据库，支持更大规模
- **Qdrant**：高性能，支持过滤和聚合
- **Weaviate**：支持混合搜索（向量 + 关键词）

**适用场景**：当数据量超过 1000 万条或需要分布式部署时考虑。

### 2.2 嵌入模型升级

**当前**：`bge-small-zh`（512 维，~100MB）

**可选升级**：
- `bge-base-zh`（768 维，~400MB）：精度更高
- `bge-large-zh`（1024 维，~1.2GB）：最高精度
- `bge-m3`（1024 维）：多语言，支持稠密+稀疏检索

**权衡**：维度越高 → 检索精度越高 → 存储/计算成本越高

### 2.3 重排序模型升级

**当前**：`bge-reranker-base`（~400MB）

**可选升级**：
- `bge-reranker-v2-m3`：多语言，精度更高
- `bge-reranker-large`：更大模型，精度更高

### 2.4 LLM 升级

**当前**：DeepSeek API（云端）

**可选方案**：
- 本地部署开源模型（Qwen、ChatGLM 等）
- 混合方案：简单问题用本地模型，复杂问题用 API

### 2.5 多模态支持

**当前**：纯文本检索

**未来**：
- 图片检索（法律条文截图、印章识别）
- 表格检索（合同条款对比表）
- 音频/视频（庭审录音、法律讲座）

### 2.6 知识图谱

**当前**：纯向量检索

**未来**：
- 构建法律知识图谱（法条关系、案例引用）
- 图谱 + 向量混合检索
- 支持复杂推理（如"哪些法条与第 X 条相关"）

| 改进项 | 优先级 | 预计收益 | 实施难度 |
|--------|--------|----------|----------|
| 图片 PDF OCR | 高 | 扩展文档支持范围 | 中 |
| 嵌入模型升级 | 中 | 提升检索精度 | 低 |
| 重排序模型升级 | 中 | 提升排序质量 | 低 |
| 知识图谱 | 低 | 支持复杂推理 | 高 |
| 向量数据库升级 | 低 | 支持更大规模 | 中 |
| 多模态支持 | 低 | 扩展应用场景 | 高 |
