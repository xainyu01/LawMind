## 下一步开发任务

### ~~任务 1：RAGAS 评测~~ ✅ 已完成

**目标**：搭建自动化评测框架，量化 RAG 系统质量。

**实现方案**：由于 `ragas` 库与 `langchain-community`（已 sunset）存在依赖冲突，采用自定义实现 RAGAS 风格评测，使用 DeepSeek 作为评判 LLM。

**文件**：
- `scripts/eval_ragas.py` — 评测脚本
- `data/eval/legal_eval_dataset.json` — 评测数据集（10 道法律题目）
- `data/eval/eval_results.json` — 评测结果

**评测指标**：
- Faithfulness（答案忠实度）：答案中的声明是否被上下文支持
- Answer Relevancy（答案相关性）：答案与问题的相关程度
- Context Precision（上下文精确度）：检索上下文中相关比例
- Context Recall（上下文召回率）：标准答案被上下文覆盖比例

**运行命令**：
```bash
HF_ENDPOINT=https://hf-mirror.com PYTHONPATH=. uv run python scripts/eval_ragas.py
HF_ENDPOINT=https://hf-mirror.com PYTHONPATH=. uv run python scripts/eval_ragas.py --verbose
```

### ~~任务 2：文件上传功能~~ ✅ 已完成

**目标**：Streamlit 界面支持上传文档并入库。

**实现**：
- `app.py` 侧边栏添加 `st.file_uploader`（支持 PDF/Word/TXT）
- 上传文件保存到 `data/docs/`
- 点击"开始入库"按钮后调用 `document_loader` 解析并入库
- 入库成功后自动重置 BM25 索引

### ~~任务 3：用户反馈收集~~ ✅ 已完成

**目标**：记录用户对回答的满意度。

**实现**：
- 每条 assistant 消息下方添加 👍/👎 按钮
- 反馈记录存入 `data/feedback.jsonl`（JSONL 格式）
- 包含时间戳、会话 ID、问题、回答、评分、意图等字段

## 注意事项

1. **GPU 操作需确认**：模型加载、训练等 GPU 操作前需二次确认
2. **HF_ENDPOINT 必须设置**：加载嵌入/重排序模型时必须有 `HF_ENDPOINT=https://hf-mirror.com`
3. **PYTHONPATH**：脚本运行时需要 `PYTHONPATH=.` 从 `app/` 导入模块
4. **PyTorch CUDA 索引**：`pyproject.toml` 中 `[tool.uv.sources]` 配置不能删除，否则会安装 CPU 版本

## 下一阶段任务（Phase 6：高级功能）

### 任务 4：LLM 意图分类

**目标**：用轻量 LLM Prompt 替换关键词方案，提升意图识别准确率。

**步骤**：
1. `app/rag/generator.py` 添加 `classify_intent_with_llm` 函数
2. 设计轻量分类 Prompt（5 种意图类型）
3. 替换现有关键词匹配逻辑
4. 测试分类准确性

### 任务 5：法条时效性检查

**目标**：元数据标注法条的颁布/废止日期，检索时自动过滤已废止法条。

**步骤**：
1. 设计时效性元数据模型（effective_date, status, repealed_date）
2. `scripts/ingest.py` 入库时标注时效性信息
3. `app/rag/vector_store.py` 添加过滤条件支持
4. `app/rag/retriever.py` 检索时过滤已废止法条

### 任务 6：监控接口

**目标**：暴露 Prometheus 指标，支持 Grafana 可视化监控。

**步骤**：
1. 安装 `prometheus_client` 依赖
2. `app/api/routes.py` 添加 `/metrics` 端点
3. 定义核心指标（请求计数、耗时、缓存命中等）
4. 集成到 Grafana 仪表板
