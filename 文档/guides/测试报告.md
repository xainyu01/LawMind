## 五、遇到的问题与解决方案

### 问题 1：Windows GBK 编码导致进度条和测试输出乱码

**现象**：控制台输出中文乱码，Unicode 字符（如 `█░`）无法显示。

**原因**：Windows 默认编码为 GBK，不支持部分 Unicode 字符。

**解决**：进度条改用 ASCII 字符 `#` 和 `-`；测试输出保持中文但避免特殊 Unicode。

### 问题 2：pytest 未安装

**现象**：`No module named pytest`。

**解决**：`uv add --dev pytest requests` 添加开发依赖。

### 问题 3：Settings 类属性在类定义时求值

**现象**：`patch.dict(os.environ, {"LLM_MODEL": "gpt-4"})` 后 `Settings().LLM_MODEL` 仍为原值。

**原因**：`Settings` 类使用 `LLM_MODEL: str = os.getenv(...)` 作为类属性，在类定义时（import 时）就已求值，后续修改 `os.environ` 不影响已定义的属性。

**解决**：修改测试逻辑，改为验证 `.env` 文件中的值是否被正确加载，而非尝试覆盖。

### 问题 4：Windows 文件删除权限错误

**现象**：`PermissionError: [WinError 32]` 删除临时文件失败。

**原因**：Windows 上 `TextLoader` 打开文件后未释放句柄，`NamedTemporaryFile` 的 `delete=False` 模式下 `os.unlink` 仍可能失败。

**解决**：改用 `tmp.close()` 后再 `os.unlink`，并用 `try/except PermissionError` 兜底。

### 问题 5：RecursiveCharacterTextSplitter 无法分段无分隔符文本

**现象**：`"A" * 1000` 只生成 1 个 chunk。

**原因**：`RecursiveCharacterTextSplitter` 依赖分隔符（`\n\n`、`\n`、`。` 等）进行分段。纯重复字符无分隔符，无法拆分。

**解决**：修改测试用例，使用包含分隔符的文本。

### 问题 6：混合检索首次查询耗时过长

**现象**：首次混合检索耗时 18.9s，后续查询 6.4s。

**原因**：首次查询需要从 ChromaDB 加载全部 136 万条文档构建 BM25 索引。后续查询从磁盘缓存加载。MMR 去重需要查询文档向量，增加额外耗时。

**说明**：这是预期行为。BM25 索引构建后持久化到 `data/chroma/bm25_cache/`，后续启动直接加载。

## 六、有待改进的地方

### 6.1 检索精度

| 问题 | 说明 | 建议 |
|------|------|------|
| 精确法条号检索不精确 | 搜索"民法典第1043条"未精确匹配 | BM25 分词对法条号处理不佳，考虑添加法条号专用索引或正则匹配预处理 |
| 混合检索耗时较长 | 后续查询仍需 6.4s | MMR 需要查询文档向量，可考虑缓存 embedding 或降低 MMR 计算频率 |

### 6.2 测试覆盖

| 缺失 | 说明 | 建议 |
|------|------|------|
| API 集成测试 | 需要启动完整服务 | 添加 pytest fixture 启动 TestClient |
| MySQL 集成测试 | auth 模块依赖 MySQL | 使用 SQLite 内存数据库替代测试 |
| 流式生成测试 | generate_stream 未测试 | 添加流式响应验证 |
| 文件上传测试 | 仅手动测试 | 添加自动化文件上传测试 |

### 6.3 性能优化

| 方向 | 说明 |
|------|------|
| BM25 索引增量更新 | 当前每次重建全量索引，可改为增量 |
| MMR 向量缓存 | 缓存常用文档的 embedding，减少 ChromaDB 查询 |
| 检索结果缓存 | 对相同或相似查询缓存检索结果 |
| 异步检索 | BM25 和向量检索可并行执行 |

### 6.4 代码质量

| 项目 | 说明 |
|------|------|
| JWT 密钥长度 | 测试中出现 `InsecureKeyLengthWarning`，当前密钥 27 字节，建议至少 32 字节 |
| langchain-community 弃用警告 | `langchain-community` 已 sunset，长期需迁移到独立集成包 |
| jieba pkg_resources 警告 | jieba 使用了弃用的 `pkg_resources` API，等待上游修复 |
