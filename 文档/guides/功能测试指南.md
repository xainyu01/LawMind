## 四、性能测试

### 4.1 检索响应时间

**测试命令**：
```bash
HF_ENDPOINT=https://hf-mirror.com PYTHONPATH=. uv run python -c "
import time
from app.rag.vector_store import search

# 预热
search('测试', top_k=1)

# 测试响应时间
queries = ['夫妻义务', '离婚条件', '故意杀人', '合同违约']
for q in queries:
    start = time.time()
    results = search(q, top_k=5)
    elapsed = time.time() - start
    print(f'{q}: {elapsed:.3f}s, 返回 {len(results)} 条')
"
```

**期望结果**：
- 单次检索 < 100ms
- 平均响应时间 < 50ms

## 七、常见问题排查

### 问题 1：检索返回空结果

**可能原因**：
- ChromaDB 数据未加载
- 嵌入模型未正确初始化

**解决方法**：
```bash
# 检查数据
PYTHONPATH=. uv run python -c "from app.rag.vector_store import get_collection_count; print(get_collection_count())"

# 重新入库
HF_ENDPOINT=https://hf-mirror.com PYTHONPATH=. uv run python scripts/ingest_auto.py --split-mode legal data/datasets/scl/texts/
```

---

### 问题 2：混合检索超时

**可能原因**：
- BM25 索引需要重建
- 数据量过大

**解决方法**：
```bash
# 清除 BM25 缓存
rm -rf data/chroma/bm25_cache/

# 重新查询（会自动重建索引）
HF_ENDPOINT=https://hf-mirror.com PYTHONPATH=. uv run python scripts/search_test.py --hybrid "测试查询"
```

---

### 问题 3：where 过滤失败

**可能原因**：
- HNSW 索引损坏
- 元数据缺失

**解决方法**：
```bash
# 重新入库
rm -rf data/chroma/
HF_ENDPOINT=https://hf-mirror.com PYTHONPATH=. uv run python scripts/ingest_auto.py --split-mode legal data/datasets/scl/texts/
```
