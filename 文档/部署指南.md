# 法律 RAG 系统 — 部署指南

本文档指导如何将法律 RAG 系统从本地开发环境部署到云服务器。

**前提**：embedding 和重排序在本地完成，服务器不需要 GPU。

---

## 一、服务器配置推荐

### 1.1 入门级（个人/测试）

| 资源 | 配置 | 说明 |
|------|------|------|
| **CPU** | 2 核 | 足够处理并发请求 |
| **内存** | 4 GB | Redis + MySQL + 应用 |
| **磁盘** | 50 GB SSD | 文档 + 数据库 + 日志 |
| **带宽** | 5 Mbps | 足够 API 响应 |

**预估月费**：¥50-100/月（阿里云/腾讯云轻量应用服务器）

### 1.2 生产级（多用户/正式使用）

| 资源 | 配置 | 说明 |
|------|------|------|
| **CPU** | 4 核 | 处理更多并发 |
| **内存** | 8 GB | Redis 缓存 + MySQL + 监控 |
| **磁盘** | 100 GB SSD | 文档增长 + 数据备份 |
| **带宽** | 10 Mbps | 多用户同时访问 |

**预估月费**：¥200-400/月

### 1.3 高可用级（企业/高并发）

| 资源 | 配置 | 说明 |
|------|------|------|
| **CPU** | 8 核 | 高并发处理 |
| **内存** | 16 GB | 充足的缓存空间 |
| **磁盘** | 200 GB SSD | 大量文档 + 备份 |
| **带宽** | 20 Mbps | 高流量场景 |

**预估月费**：¥500-1000/月

### 1.4 磁盘空间估算

| 组件 | 占用 | 说明 |
|------|------|------|
| Docker 镜像 | ~2 GB | 应用 + Redis + MySQL + Nginx |
| ChromaDB 数据 | ~1 GB | 向量数据（本地打包上传） |
| MySQL 数据 | ~500 MB | 对话记录、日志 |
| MinIO 文档 | 取决于上传量 | 原始文档存储 |
| Prometheus 数据 | ~1 GB | 15 天监控数据 |
| 日志文件 | ~500 MB | 应用日志 |

**总计**：约 5-10 GB 基础占用，建议预留 50GB+

### 1.5 推荐云服务商

| 服务商 | 推荐产品 | 说明 |
|--------|----------|------|
| 阿里云 | 轻量应用服务器 / ECS | 国内稳定，文档齐全 |
| 腾讯云 | 轻量应用服务器 | 性价比高，学生优惠 |
| 华为云 | ECS | 企业级稳定 |
| AWS | EC2 (海外) | 海外用户选择 |

**操作系统**：推荐 Ubuntu 22.04 LTS / Debian 12 / CentOS Stream 9

## 二、本地准备

### 2.1 完成 Embedding

在本地完成所有文档的 embedding 和入库：

```bash
# 确保本地环境正常
cd C:\Users\Admin\Desktop\RAG

# 入库法律文档
HF_ENDPOINT=https://hf-mirror.com PYTHONPATH=. uv run python scripts/ingest.py data/docs/

# 验证入库数量
HF_ENDPOINT=https://hf-mirror.com PYTHONPATH=. uv run python -c "
from app.rag.vector_store import VectorStore
vs = VectorStore()
print(f'入库文档数: {vs.collection.count()}')
"
```

### 2.2 打包向量数据

```bash
# 打包 ChromaDB 数据
cd C:\Users\Admin\Desktop\RAG
tar -czf chroma-data.tar.gz data/chroma/

# 打包 BM25 索引缓存（如果有）
tar -czf bm25-cache.tar.gz data/cache/

# 打包文档数据
tar -czf docs-data.tar.gz data/docs/
```

### 2.3 准备环境变量

创建 `.env.production` 文件：

```bash
# DeepSeek API 配置
DEEPSEEK_API_KEY=your-api-key-here
DEEPSEEK_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat

# 嵌入模型配置（本地已完成，服务器不需要加载模型）
EMBEDDING_MODEL_NAME=BAAI/bge-small-zh
HF_ENDPOINT=https://hf-mirror.com

# 认证配置
AUTH_ENABLED=true
AUTH_MODE=api_key
API_KEYS=["your-production-api-key"]
JWT_SECRET=your-jwt-secret-here
JWT_EXPIRE_HOURS=24

# 限流配置
RATE_LIMIT_ENABLED=true
RATE_LIMIT_PER_MINUTE=60
RATE_LIMIT_PER_HOUR=1000

# MinIO 配置
MINIO_ENDPOINT=minio:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=your-minio-secret
MINIO_SECURE=false
MINIO_BUCKET=legal-documents

# MySQL 配置
MYSQL_URL=mysql+pymysql://root:your-mysql-password@mysql:3306/legal_rag

# Redis 配置
REDIS_URL=redis://redis:6379/0

# Grafana 配置
GF_SECURITY_ADMIN_PASSWORD=your-grafana-password
```

## 三、服务器初始化

### 3.1 连接服务器

```bash
# SSH 连接
ssh root@your-server-ip

# 或使用密钥
ssh -i ~/.ssh/id_rsa root@your-server-ip
```

### 3.2 系统更新

```bash
# Ubuntu/Debian
apt update && apt upgrade -y

# CentOS Stream
dnf update -y
```

### 3.3 安装 Docker

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

# 安装 Docker Compose
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

### 3.4 创建项目目录

```bash
mkdir -p /opt/legal-rag
cd /opt/legal-rag
```

## 五、配置调整

### 5.1 修改 docker-compose.yml

确保 `docker-compose.yml` 中的配置适合生产环境：

```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "8000:8000"
    env_file:
      - .env
    volumes:
      - ./data:/app/data
    depends_on:
      - redis
      - mysql
      - minio
    restart: always
    deploy:
      resources:
        limits:
          memory: 2G

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    restart: always
    command: redis-server --appendonly yes --maxmemory 512mb --maxmemory-policy allkeys-lru

  mysql:
    image: mysql:8.0
    ports:
      - "3306:3306"
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD:-password}
      MYSQL_DATABASE: legal_rag
    volumes:
      - mysql-data:/var/lib/mysql
      - ./scripts/init.sql:/docker-entrypoint-initdb.d/init.sql
    restart: always

  minio:
    image: minio/minio:latest
    ports:
      - "9000:9000"
      - "9001:9001"
    environment:
      MINIO_ROOT_USER: ${MINIO_ACCESS_KEY:-minioadmin}
      MINIO_ROOT_PASSWORD: ${MINIO_SECRET_KEY:-minioadmin}
    volumes:
      - minio-data:/data
    command: server /data --console-address ":9001"
    restart: always

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./ssl:/etc/nginx/ssl:ro
    depends_on:
      - app
    restart: always

  prometheus:
    image: prom/prometheus:latest
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
      - prometheus-data:/prometheus
    restart: always

  grafana:
    image: grafana/grafana:latest
    ports:
      - "3000:3000"
    environment:
      GF_SECURITY_ADMIN_PASSWORD: ${GF_SECURITY_ADMIN_PASSWORD:-admin}
    volumes:
      - grafana-data:/var/lib/grafana
      - ./grafana/provisioning:/etc/grafana/provisioning
      - ./grafana/dashboards:/var/lib/grafana/dashboards
    restart: always

volumes:
  redis-data:
  mysql-data:
  minio-data:
  prometheus-data:
  grafana-data:
```

### 5.2 修改 Nginx 配置

修改 `nginx.conf` 适配生产环境：

```nginx
events {
    worker_connections 1024;
}

http {
    # 限流配置
    limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;

    # 上游服务
    upstream app {
        server app:8000;
    }

    server {
        listen 80;
        server_name your-domain.com;  # 替换为你的域名

        # HTTP 重定向到 HTTPS（可选）
        # return 301 https://$server_name$request_uri;

        # API 代理
        location /api/ {
            limit_req zone=api burst=20 nodelay;
            proxy_pass http://app;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_read_timeout 120s;
        }

        # WebSocket 代理
        location /ws/ {
            proxy_pass http://app;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
            proxy_read_timeout 86400;
        }

        # 健康检查
        location /health {
            proxy_pass http://app;
        }

        # 监控端点（仅内网访问）
        location /metrics {
            allow 10.0.0.0/8;
            allow 172.16.0.0/12;
            allow 192.168.0.0/16;
            deny all;
            proxy_pass http://app;
        }

        # MinIO 控制台（可选）
        location /minio/ {
            proxy_pass http://minio:9001/;
        }

        # Grafana（可选）
        location /grafana/ {
            proxy_pass http://grafana:3000/;
        }
    }

    # HTTPS 配置（可选）
    # server {
    #     listen 443 ssl;
    #     server_name your-domain.com;
    #
    #     ssl_certificate /etc/nginx/ssl/cert.pem;
    #     ssl_certificate_key /etc/nginx/ssl/key.pem;
    #
    #     location / {
    #         proxy_pass http://app;
    #     }
    # }
}
```

### 8.1 常用命令


### 8.2 数据备份

创建备份脚本 `scripts/backup.sh`：

```bash
#!/bin/bash
BACKUP_DIR="/opt/legal-rag/backups"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份 MySQL
docker-compose exec mysql mysqldump -u root -p legal_rag > $BACKUP_DIR/mysql_$DATE.sql

# 备份 ChromaDB
tar -czf $BACKUP_DIR/chroma_$DATE.tar.gz data/chroma/

# 备份 MinIO 数据
tar -czf $BACKUP_DIR/minio_$DATE.tar.gz data/minio/

# 删除 30 天前的备份
find $BACKUP_DIR -name "*.sql" -mtime +30 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

echo "备份完成: $BACKUP_DIR"
```

设置定时备份：

```bash
# 添加执行权限
chmod +x scripts/backup.sh

# 添加到 crontab（每天凌晨 3 点备份）
crontab -e
# 添加：0 3 * * * /opt/legal-rag/scripts/backup.sh >> /var/log/backup.log 2>&1
```

### 8.3 日志管理

创建日志轮转配置 `/etc/logrotate.d/legal-rag`：

```bash
/opt/legal-rag/logs/*.log {
    daily
    rotate 14
    compress
    delaycompress
    missingok
    notifempty
    create 0644 root root
}
```

### 8.4 监控告警

**Grafana 告警配置**：

1. 访问 `http://your-server-ip:3000`
2. 登录（默认 admin / 你在 .env 中设置的密码）
3. 添加 Prometheus 数据源：`http://prometheus:9090`
4. 导入仪表盘或创建自定义告警规则

**推荐告警规则**：
- API 响应时间 > 5 秒
- 错误率 > 5%
- 内存使用 > 80%
- 磁盘使用 > 90%

## 九、安全加固

### 9.1 防火墙配置

```bash
# Ubuntu/Debian (ufw)
ufw allow 22/tcp    # SSH
ufw allow 80/tcp    # HTTP
ufw allow 443/tcp   # HTTPS
ufw deny 3306/tcp   # 禁止外部访问 MySQL
ufw deny 6379/tcp   # 禁止外部访问 Redis
ufw deny 9000/tcp   # 禁止外部访问 MinIO
ufw enable

# CentOS (firewalld)
firewall-cmd --permanent --add-service=ssh
firewall-cmd --permanent --add-service=http
firewall-cmd --permanent --add-service=https
firewall-cmd --reload
```

### 9.2 SSH 安全

```bash
# 编辑 SSH 配置
vim /etc/ssh/sshd_config

# 修改以下配置：
# PermitRootLogin no           # 禁止 root 登录
# PasswordAuthentication no    # 禁止密码登录，仅允许密钥
# Port 2222                    # 修改默认端口

# 重启 SSH
systemctl restart sshd
```

### 9.3 Docker 安全

```bash
# 限制容器资源
# 在 docker-compose.yml 中已配置 deploy.resources.limits

# 定期更新镜像
docker-compose pull
```

## 十、故障排查

### 10.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 服务启动失败 | 端口被占用 | `lsof -i :8000` 查看并 kill |
| MySQL 连接失败 | 密码错误或未启动 | 检查 .env 和 docker-compose logs mysql |
| Redis 连接失败 | 内存不足 | 检查 Redis 配置和服务器内存 |
| 向量检索无结果 | 数据未上传 | 确认 data/chroma/ 目录存在且有数据 |
| API 返回 401 | API Key 错误 | 检查 .env 中的 API_KEYS 配置 |
| 响应时间长 | 网络或 LLM 延迟 | 检查 DeepSeek API 连通性 |

### 10.2 日志查看

```bash
# 应用日志

# Nginx 访问日志
```
