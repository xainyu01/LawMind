## 三、数据集入库

### SCL 数据集

- **来源**：`twang2218/chinese-law-and-regulations`（HuggingFace）
- **格式**：Parquet（`train-00000-of-00001-*.parquet`）
- **规模**：22552 部中国法律法规
- **转换**：Parquet → 22552 个 .txt 文件（`data/datasets/scl/texts/`）
- **入库**：使用 `--split-mode legal`（按"第X条"分段）
- **结果**：367369 个文档片段入库 ChromaDB

**转换脚本**：
```python
import pandas as pd
import os

df = pd.read_parquet("data/datasets/scl/data/train-00000-of-00001-xxx.parquet")
os.makedirs("data/datasets/scl/texts", exist_ok=True)
for idx, row in df.iterrows():
    title = row["title"]
    content = row["content"]
    with open(f"data/datasets/scl/texts/{idx}.txt", "w", encoding="utf-8") as f:
        f.write(f"{title}\n\n{content}")
```

## 四、遇到的问题与修复

### 问题 1：SCL Parquet 格式无法直接入库

**现象**：`ingest.py` 只支持 PDF/DOCX/TXT，不支持 Parquet。

**原因**：SCL 数据集使用 HuggingFace 标准的 Parquet 格式分发。

**修复**：编写 Python 脚本将 Parquet 转换为 TXT 文件。

---

### 问题 2：Windows 文件名编码错误

**现象**：中文文件名导致 `OSError`。

**原因**：Windows 文件系统对中文字符支持有限。

**修复**：使用数字索引作为文件名（`0.txt`, `1.txt`, ...）。

---

### 问题 3：get_all_documents() SQL 变量溢出

**现象**：
```
chromadb.errors.InternalError: Error executing plan: Internal error:
error returned from database: (code: 1) too many SQL variables
```

**原因**：一次获取 619K+ 文档时，SQLite 参数化查询超出变量限制。

**修复**：修改为分批获取（每批 5000 条）。

**修改文件**：`app/rag/vector_store.py` 第 79-95 行

---

### 问题 4：HNSW 索引损坏（核心问题）

**现象**：
```
chromadb.errors.InternalError: Error executing plan:
Error constructing hnsw segment reader:
Error deserializing pickle file: eval error at offset 29609312: trailing bytes found
```

**原因**：SCL 入库（22552 个文件）过程中，HNSW 索引的 `index_metadata.pickle` 文件被截断。可能是进程被 OOM Killer 终止或手动杀死。

**临时修复**：删除损坏的 pickle 文件，`col.get()` 恢复正常。

**遗留问题**：`col.query()` 在混合检索流程中仍报错 `Error finding id`，需要重建索引。

**详见**：`文档/2026-07-12_ChromaDB索引损坏与数据完整性问题.md`

---

### 问题 5：维度不匹配（384 vs 512）

**现象**：
```
Collection expecting embedding with dimension of 512, got 384
```

**原因**：ChromaDB 默认嵌入函数 `all-MiniLM-L6-v2` 产生 384 维向量，而我们的 `bge-small-zh` 产生 512 维向量。直接调用 `col.query(query_texts=["..."])` 会使用默认函数。

**解决**：我们的代码始终传入 `query_embeddings`（由 bge-small-zh 生成），不依赖 ChromaDB 默认函数。此问题仅在测试时出现。
