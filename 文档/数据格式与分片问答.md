## 一、Parquet 格式

### Q: Parquet 是什么？

Apache Parquet 是一种**列式存储**的二进制文件格式，专为大数据分析设计。

```
传统 CSV/JSON（行式存储）：          Parquet（列式存储）：
┌──────────────────────────┐     ┌──────────────────────────┐
│ id | name  | age | city  │     │ id:  [1, 2, 3, 4, ...]   │
│ 1  | 张三  | 25  | 北京  │     │ name: [张三, 李四, ...]   │
│ 2  | 李四  | 30  | 上海  │     │ age:  [25, 30, 28, ...]   │
│ 3  | 王五  | 28  | 广州  │     │ city: [北京, 上海, ...]    │
└──────────────────────────┘     └──────────────────────────┘
读取 name 列 → 扫描所有行           读取 name 列 → 直接定位列块
```

### Q: 什么时候使用 Parquet？

| 场景 | 使用 Parquet | 使用 JSON |
|------|-------------|-----------|
| HuggingFace 数据集分发 | ✅ 默认格式 | 可选 |
| 大规模数据分析（Spark/Pandas） | ✅ 首选 | 不适合 |
| LLM 训练数据 | 有时用 | ✅ 更常见 |
| API 数据交换 | ❌ 不适合 | ✅ 首选 |
| 配置文件 | ❌ | ✅ 首选 |
| 流式读取 | ❌ | ✅ 逐行读取 |

### Q: 为什么 HuggingFace 用 Parquet？

1. **压缩率高**：列式存储 + 压缩算法，文件体积比 JSON 小 50-80%
2. **读取快**：只读需要的列，不用解析整行
3. **类型安全**：schema 内嵌，不会出现 "age" 字段有时是数字有时是字符串
4. **生态兼容**：Pandas、Spark、DuckDB 等直接读取

```python
# Parquet 读取示例
import pandas as pd
df = pd.read_parquet("data.parquet")          # 一行代码
# 只读某一列（极快）
titles = pd.read_parquet("data.parquet", columns=["title"])
```

### Q: Parquet vs JSON 对比

| 特性 | Parquet | JSON |
|------|---------|------|
| 存储方式 | 列式、二进制 | 行式、文本 |
| 文件大小 | 小（压缩率高） | 大（无压缩） |
| 读取速度 | 快（列裁剪） | 慢（全量解析） |
| 类型信息 | 有 schema | 无 schema |
| 可读性 | 二进制，不可直接阅读 | 文本，可直接阅读 |
| 流式处理 | 不支持 | ✅ 逐行读取 |
| 适用规模 | GB~TB 级 | KB~MB 级 |

## 二、大模型各阶段的数据格式

### 训练阶段

| 阶段 | 常用格式 | 原因 |
|------|---------|------|
| 预训练（Pre-training） | JSONL / Parquet / 纯文本 | 海量数据，需要高效读取 |
| 指令微调（SFT） | JSON / JSONL | 结构化指令-回答对 |
| RLHF 偏好数据 | JSONL | 每行一条 preference pair |
| 评测数据集 | JSON / JSONL / Parquet | 取决于发布者习惯 |

```
# JSONL（每行一个 JSON 对象，流式读取最友好）
{"instruction": "什么是诉讼时效？", "output": "诉讼时效是指..."}
{"instruction": "民法典第1043条？", "output": "夫妻应当互相忠实..."}

# JSON（完整数组）
[
  {"instruction": "什么是诉讼时效？", "output": "..."},
  {"instruction": "民法典第1043条？", "output": "..."}
]
```

### RAG 系统阶段

| 阶段 | 常用格式 | 本项目使用 |
|------|---------|-----------|
| 原始文档 | PDF / Word / TXT | ✅ `data/docs/` |
| 数据集分发 | Parquet / JSONL | ✅ `data/datasets/` (SCL=Parquet, LeCaRD=JSONL) |
| 分段后入库 | ChromaDB 内部格式 | ✅ `data/chroma/` |
| 检索结果 | dict / JSON | ✅ 内存传递 |

## 三、本项目的分片（Chunking）逻辑

### 两种分段模式

#### 1. 默认模式（`--split-mode default`）

使用 LangChain `RecursiveCharacterTextSplitter`，按递归分隔符切割：

```
chunk_size = 500 字符
chunk_overlap = 50 字符（前后片段重叠，避免语义断裂）

分隔符优先级：
  \n\n（段落） → \n（换行） → 。（句号） → ；（分号） → ，（逗号） → 空格
```

**原理**：先尝试按段落切，如果一段超过 500 字，再按句号切，依此类推。

```
原文（1200字）→ 切成 3 段：
  [段1: 0-500字] [段2: 450-950字] [段3: 900-1200字]
                 ↑ 与段1重叠50字    ↑ 与段2重叠50字
```

#### 2. 法律模式（`--split-mode legal`）

按法条结构精确分段，保持法条完整性：

```python
# 法律法规：按"第X条"分段
正则: r'(第[一二三四五六七八九十百千零\d]+条)'

# 判决书：按关键段落分段
正则: r'(?=本院认为|判决如下|如不服本判决|依照)'

# 合同：按"第X条"分段（同法律法规）
```

**示例**：

```
原文：
  第一千零四十二条 禁止包办、买卖婚姻和其他干涉婚姻自由的行为...
  第一千零四十三条 夫妻应当互相忠实，互相尊重，互相关爱...

分段结果：
  片段1: "第一千零四十二条 禁止包办、买卖婚姻..." (article=第一千零四十二条)
  片段2: "第一千零四十三条 夫妻应当互相忠实..."   (article=第一千零四十三条)
```

### 分段后每个片段的元数据

```python
{
    "source": "民法典节选.txt",      # 来源文件
    "chunk_index": 42,               # 片段序号
    "article": "第一千零四十三条",    # 法条编号（法律模式）
    "doc_type": "law",               # 文档类型
    "effective_date": "2021-01-01",  # 生效日期
    "status": "active"               # 时效状态（active/repealed）
}
```

### 为什么 chunk_size=500？

| chunk_size | 优点 | 缺点 |
|-----------|------|------|
| 200 | 精确匹配 | 语义不完整，丢失上下文 |
| **500** | **平衡精确度和完整性** | - |
| 1000 | 上下文丰富 | 检索时噪音多，embedding 质量下降 |
| 2000+ | 完整文档 | 向量检索基本失效 |

法律条文通常 100-300 字/条，500 字可以完整包含一条法条 + 部分上下文。

## 四、SCL 数据集格式转换

SCL 数据集使用 Parquet 格式（HuggingFace 标准），入库前需要转换：

```python
import pandas as pd
import os

df = pd.read_parquet("data/datasets/scl/data/train-00000-of-00001-xxx.parquet")
# 列：title, content, publish_date, status, office, ...

os.makedirs("data/datasets/scl/texts", exist_ok=True)
for idx, row in df.iterrows():
    title = row["title"]
    content = row["content"]
    with open(f"data/datasets/scl/texts/{idx}.txt", "w", encoding="utf-8") as f:
        f.write(f"{title}\n\n{content}")

# 然后入库
# uv run python scripts/ingest.py --split-mode legal data/datasets/scl/texts/
```
