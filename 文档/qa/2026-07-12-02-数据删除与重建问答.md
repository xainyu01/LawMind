## 三、UV 与 PyTorch/CUDA

### Q: UV 管理依赖是在 UV 中已经安装 PyTorch 吗？

**是的**，UV 会将 PyTorch 安装到项目的虚拟环境中：

```
安装位置：C:\Users\Admin\Desktop\RAG\.venv\Lib\site-packages\torch\

安装过程：
1. 读取 pyproject.toml 中的 dependencies: ["torch>=2.11.0"]
2. 根据 [tool.uv.sources] 配置，从 pytorch-cu128 索引下载
3. 安装到 .venv 虚拟环境
```

### Q: CUDA 版本是本身的版本吗？

**不是**，PyTorch 自带 CUDA 运行时，不需要系统安装 CUDA Toolkit：

```
系统要求：
├── NVIDIA GPU 驱动（已安装：RTX 5060）
│   └── 支持 CUDA 12.0+
│
└── CUDA Toolkit（不需要安装）
    └── PyTorch 自带 CUDA 12.8 运行时

pyproject.toml 配置：
[[tool.uv.index]]
name = "pytorch-cu128"
url = "https://download.pytorch.org/whl/cu128"  ← CUDA 12.8 版本

[tool.uv.sources]
torch = { index = "pytorch-cu128" }  ← 从这个索引下载
```

**验证命令**：
```bash
# 检查 PyTorch 是否能使用 GPU
PYTHONPATH=. uv run python -c "import torch; print(torch.cuda.is_available())"

# 检查 PyTorch 内置的 CUDA 版本
PYTHONPATH=. uv run python -c "import torch; print(torch.version.cuda)"
```

## 五、备份建议

### Q: 需要备份吗？

**不需要备份**，原因：

1. **源数据集保留**：`data/datasets/` 和 `data/docs/` 不会被删除
2. **损坏的数据**：当前的 ChromaDB 数据已经损坏，备份无意义
3. **可重新生成**：所有向量都可以从源文件重新计算

### 如果想备份 BM25 缓存

BM25 缓存可以加速首次启动，但会随 ChromaDB 重建而失效：

```bash
# 备份 BM25 缓存（可选）
cp -r data/chroma/bm25_cache/ data/bm25_cache_backup/

# 重新入库后，BM25 缓存会自动重建
```
